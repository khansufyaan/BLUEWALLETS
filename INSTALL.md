# Blue Wallets — Production Deployment Guide

## Prerequisites

- **Docker Engine** 24+ and **Docker Compose** v2
- **Luna HSM** — DPoD cloud partition or Luna Network HSM with PKCS#11 client installed
- **PostgreSQL** — runs as a Docker container (included)
- **mTLS Certificates** — auto-generated by included script
- Server with Linux x86_64 (the Luna PKCS#11 library is a Linux binary)

## Quick Start (5 minutes)

```bash
# 1. Clone the repository
git clone https://github.com/khansufyaan/BLUEWALLETS.git
cd BLUEWALLETS

# 2. Generate mTLS certificates (Driver <-> Console secure channel)
cd certs && bash generate-certs.sh && cd ..

# 3. Create environment file
cat > .env << 'EOF'
POSTGRES_PASSWORD=<strong-password-here>
INTERNAL_AUTH_KEY=<random-64-character-string>
HSM_PIN=<your-luna-partition-user-pin>
ETH_RPC_URL=https://eth-mainnet.g.alchemy.com/v2/<your-key>
ETH_CHAIN_ID=1
EOF

# 4. Start all services
docker compose -f docker-compose.luna.yml up -d

# 5. Wait for all containers to be healthy
docker compose -f docker-compose.luna.yml ps
```

## What Gets Deployed

Three Docker containers on two isolated networks:

```
┌─────────────────────────────────────────────────────────┐
│ SECURE ZONE (internal network — no internet)            │
│                                                         │
│  ┌──────────────┐     ┌──────────────────────┐          │
│  │  PostgreSQL   │────▶│    Blue Driver        │         │
│  │  :5432        │     │    :3100 (dashboard)  │         │
│  │  (persistent) │     │    :3200 (mTLS API)   │         │
│  └──────────────┘     │    + Luna HSM (PKCS#11)│         │
│                        └──────────┬─────────────┘        │
│                                   │ mTLS                 │
├───────────────────────────────────┼─────────────────────┤
│ DMZ (internet access)             │                      │
│                        ┌──────────▼─────────────┐        │
│                        │    Blue Console         │       │
│                        │    :3300 (bank API)     │       │
│                        │    :3400 (ops dashboard)│       │
│                        │    + RPC nodes          │       │
│                        │    + Compliance APIs    │       │
│                        └────────────────────────┘        │
└─────────────────────────────────────────────────────────┘
```

| Port | Service | Purpose | Who accesses it |
|------|---------|---------|-----------------|
| 3100 | Blue Driver Dashboard | Key ceremony, HSM config, users | Security officer |
| 3200 | Blue Driver Internal API | mTLS-secured Console-to-Driver channel | Console only (automated) |
| 3300 | Blue Console Bank API | Wallet creation, transfers, policies | Bank applications |
| 3400 | Blue Console Ops Dashboard | Daily operations, monitoring | Operations team |
| 5432 | PostgreSQL | Wallet data, transactions, audit logs | Driver only (internal) |

## Step-by-Step Setup

### Step 1: Environment Configuration

Create a `.env` file in the project root:

```bash
# REQUIRED
POSTGRES_PASSWORD=<strong-password>     # PostgreSQL password
INTERNAL_AUTH_KEY=<random-64-chars>    # Shared key for Driver-Console auth
HSM_PIN=<luna-user-pin>                # Luna HSM Crypto User PIN

# OPTIONAL — Blockchain RPC endpoints
ETH_RPC_URL=https://eth-mainnet.g.alchemy.com/v2/<key>
ETH_CHAIN_ID=1
BSC_RPC_URL=https://bsc-dataseed.binance.org
POLYGON_RPC_URL=https://polygon-rpc.com
```

### Step 2: mTLS Certificates

The `certs/` directory must contain:

| File | Purpose |
|------|---------|
| `ca.pem` | Self-signed CA certificate |
| `ca-key.pem` | CA private key |
| `driver-cert.pem` | Driver server certificate (SAN: blue-driver, localhost) |
| `driver-key.pem` | Driver server private key |
| `console-cert.pem` | Console client certificate |
| `console-key.pem` | Console client private key |

**Generate fresh certificates:**
```bash
cd certs && bash generate-certs.sh && cd ..
```

The included script generates EC P-256 certificates valid for 365 days. **Rotate annually.**

### Step 3: Luna HSM Client

The Blue Driver Docker image includes the Luna PKCS#11 client at `/opt/lunaclient/`. The following files from your Luna DPoD service must be placed in the `luna-client/` directory before building:

| File | Purpose |
|------|---------|
| `Chrystoki.conf` | PKCS#11 configuration (partition, endpoints, auth) |
| `partition-ca-certificate.pem` | Partition CA cert |
| `partition-certificate.pem` | Partition client cert |
| `server-certificate.pem` | DPoD server verification cert |
| `cvclient-min.tar` | Luna client binaries (provided by Thales) |

These files are bundled into the Docker image at build time.

### Step 4: Start the Stack

```bash
# Build and start (first time)
docker compose -f docker-compose.luna.yml up --build -d

# Check all services are healthy
docker compose -f docker-compose.luna.yml ps

# Expected output:
# NAME                  STATUS
# blue-postgres-1       Up (healthy)
# blue-blue-driver-1    Up (healthy)
# blue-blue-console-1   Up (healthy)
```

### Step 5: Key Ceremony

1. Open **http://<server>:3100** (Blue Driver Dashboard)
2. Login: `admin` / `Admin1234!` (change password immediately)
3. **Step 1 — Connect HSM**: Select Luna HSM, verify connection
4. **Step 2 — Generate Master Key**: Creates `blue:wrap:v1` (AES-256, non-extractable)
5. **Step 3 — HD Master Seed** (optional): Generates 24-word BIP-39 mnemonic for hierarchical wallets
   - **Write down the 24 words** — they are shown ONCE and never stored
   - Split via Shamir's Secret Sharing for custody
6. **Step 4 — Verification**: Confirms database, HSM, and key generation are working

### Step 6: Console Operations

1. Open **http://<server>:3400** (Blue Console Ops Dashboard)
2. Login with same credentials
3. Navigate to **API Keys** to create keys for bank applications
4. Navigate to **Settings** to configure RPC endpoints and compliance APIs

## Using Pre-Built Images (Alternative to Building)


The `docker-compose.client.yml` references GHCR images instead of building locally.

## Wallet Key Modes

Blue Wallets supports two key management modes, auto-detected at startup:

### Legacy Mode (HSM Token Keys)
- Each wallet gets an independent EC keypair generated inside the HSM
- Private keys are permanent HSM objects (`CKA_EXTRACTABLE=false`)
- One HSM slot consumed per wallet
- Recovery: Only via Luna HA backup

### HD Mode (BIP-32 Derived + AES Encrypted)
- One master seed on the HSM, child keys derived per BIP-44 path
- Child keys AES-256 encrypted by HSM, stored in PostgreSQL
- Zero HSM slots consumed per wallet — scales to millions
- Recovery: 24-word mnemonic recovers all wallets

HD mode activates automatically when the HD Master Seed ceremony (Step 3) is completed.

## Security Architecture

| Layer | Protection |
|-------|-----------|
| HSM key material | FIPS 140-3 Level 3 — private keys never leave HSM boundary |
| Driver ↔ Console | Mutual TLS with client certificates |
| Bank API auth | API keys (SHA-256 hashed, permission-scoped) |
| User auth | WebAuthn passkeys (primary) + bcrypt passwords |
| Policy enforcement | Per-wallet rules evaluated before HSM signing |
| Compliance | TRM Labs + Chainalysis KYT + Notabene Travel Rule (fail-closed) |
| Audit trail | Hash-chained, HMAC-signed immutable logs |
| Database | AES-256 encrypted wallet keys, plaintext balance only |

## Backup & Recovery

### Database
```bash
# Backup PostgreSQL
docker exec blue-postgres-1 pg_dump -U blue bluewallets > backup.sql

# Restore
docker exec -i blue-postgres-1 psql -U blue bluewallets < backup.sql
```

### HSM Keys
- `blue:wrap:v1` — backed up via Luna HA cluster replication
- `blue:encrypt:v1` — backed up via Luna HA cluster replication
- `blue:hd:master:v1` — backed up via Luna HA + 24-word mnemonic (Shamir split)

### Certificates
Store `certs/ca-key.pem` securely — it signs all mTLS certificates. Losing it requires regenerating all certs and restarting all services.

## Troubleshooting

| Symptom | Cause | Fix |
|---------|-------|-----|
| Driver: `CKR_PIN_EXPIRED` | Luna PIN expired | Change PIN in Driver → HSM Health → Change PIN |
| Console: `ECONNREFUSED :3200` | Driver not ready | Wait for Driver healthcheck, or restart Console |
| Console: `ERR_SSL_TLSV13_ALERT_CERTIFICATE_REQUIRED` | mTLS cert missing | Verify `certs/` directory is mounted in both containers |
| `BigInt serialization` | Missing `.toString()` on balance | Update to latest image version |
| HD wallet creation fails | HSM partition blocks key import | Normal — falls back to Strategy B (HSM encrypt + software sign) |
| Blank Console page | Auth timeout | Refresh browser, check Driver is running |
